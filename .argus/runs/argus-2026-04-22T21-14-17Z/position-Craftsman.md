## Craftsman — Round 1 Position

**Claim:** Montauk cannot become the trustworthy TECL decision factory named in the North Star until it stops describing one validation system and running another, because its core trust boundary is currently a semantic counterfeit: the code has split leaderboard admission from champion certification, but the repo still talks about `gate`, `PASS`, `promotion_ready`, and `backtest_certified` as if they all belong to one clean model.

**Evidence:** The most important thing I discovered is that Gate 7 already implements a two-stage truth the repo has not named honestly. In `scripts/validation/pipeline.py`, `promotion_ready` is set from the PASS verdict while `backtest_certified` is stricter and depends on certification checks that include `artifact_completeness`, which begins life as pending. `scripts/search/evolve.py` then admits leaderboard entries without requiring that artifact-completeness check, and the current `spike/leaderboard.json` proves the point: admitted rows exist with `promotion_ready: true` and `backtest_certified: false`. That is a coherent watchlist-plus-certification model. The problem is that `docs/validation-philosophy.md`, `docs/pipeline.md`, `docs/project-status.md`, and `CLAUDE.md` still export incompatible definitions, while marker and cross-asset are still called gates even after the code demoted them into diagnostics and score inputs. If you read this system cold, you build the wrong mental model first, exactly where the project is trying to manufacture trust.

**Biggest opportunity:** Collapse the vocabulary to match the real architecture: one explicit admission contract for leaderboard memory, one explicit certification contract for the deployable champion, and diagnostic names for checks that no longer veto. That would move Montauk directly toward the North Star because the evidence machine would finally describe its own evidence honestly.

**Biggest blocker:** The blocker is not weak math; it is contradictory authority. As long as `scripts/validation/pipeline.py`, `spike/leaderboard.json`, and the docs teach different meanings for the same words, every future improvement to validation, ranking, or certification will compound confusion instead of trust.
