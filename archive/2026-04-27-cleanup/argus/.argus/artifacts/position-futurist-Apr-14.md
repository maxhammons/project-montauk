## Futurist — Round 1 Position
**Claim:** The string-template architecture in `pine_generator.py` is an architectural dead end that will completely block the North Star goal of true automated strategy discovery within 18 months.
**Evidence:** Trace this forward: the codebase's North Star explicitly demands "automated strategy discovery," but currently, the Python backtester and the Pine Script generator maintain parallel, hardcoded logic. Because there is no shared Abstract Syntax Tree (AST) or Domain Specific Language (DSL) defining the indicator topology, the evolutionary engine is structurally limited to just tuning parameters for human-written templates. When the engine inevitably needs to discover novel logic—like combining an RSI entry with a TEMA exit—a human will have to manually write both the Python and Pine representations. This dual-maintained string-formatting architecture means the system cannot autonomously evolve new strategies.
**Biggest opportunity:** Introducing a shared configuration object or DSL that represents strategy logic abstractly. If both the Python backtester and Pine generator compile from the same fundamental source of truth, the system can autonomously mutate indicator topologies, shifting from parameter optimization to genuine programmatic logic discovery.
**Biggest blocker:** The engine's evolutionary state is locked into the raw file bytes (`_ENGINE_HASH` in `evolve.py`) and stored in a single, non-concurrent `hash-index.json` file. Any structural refactoring toward a shared AST will instantly invalidate thousands of hours of historical compute cache, and the single-file JSON index guarantees the team cannot scale compute horizontally to rebuild that history or explore the expanded search space.
