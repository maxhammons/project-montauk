# Montauk 4.0 — Beyond TECL: Multi-Asset & a Wider Action Space

**What this folder is.** The home for the *next* expansion after Montauk 3.0: trading
**beyond TECL long/flat** — both a **wider action space** (a defensive SGOV leg,
position sizing, inverse/short legs) and a **multi-asset / sector-rotation fleet**
(per-sector machines + a rotation brain). Split out of Montauk 3.0 on 2026-06-17 to
keep 3.0 cleanly single-asset.

**Status: DEFERRED / VISION.** Montauk 4.0 does not begin until 3.0 — the TECL-only
always-on factory with a bulletproof validation engine — is real. 4.0 is the honest
answer to the one thing 3.0's rigor *cannot* fix: the **data-scarcity ceiling** (~3–4
independent macro regimes on a single asset). More independent regimes — across assets
— is the only real source of the missing forward evidence. So 4.0 is where genuine
board diversity and stronger statistical power become reachable, **on the same fixed
Gold bar.**

## Why it's a separate release

Both expansions break the same 3.0 non-negotiable ("TECL-only, long/flat") and need
the same heavy lifts: a charter amendment to the asset/action universe, a redefined
benchmark + goal function, new engine position states (each needing a fresh Layer-1
integrity pass), and — for rotation — a *separately validated* overlay that is itself
notoriously overfit-prone. None of that should touch the 3.0 build.

## Hard dependency on 3.0

4.0 inherits 3.0 wholesale: the appliance / deterministic-churner principle, the
staging → manual-admission gate, and above all the **bulletproof validation engine**.
The Gold bar does not move; 4.0 just gives more runners real equipment and more
independent data to prove themselves on. See `../Montauk 3.0/` (start with its
`charter.md` and `validation-engine-hardening.md`).
