# Glossary

Project-specific vocabulary. If "we call users 'operators'", it lives here.

Populated automatically by the prompt-submit hook when the user defines terms ("we call X 'Y'", "by Z we mean…"). Seeded at setup time from definitions in raw docs if a seed source was provided.

Entries follow the format defined in `README.md`. The `Statement` field contains both the term and its definition, e.g. "**Operator**: An end-user with admin authority over a fund view."

Sorted alphabetically by term when practical. `spirit-audit` can resort on demand.
