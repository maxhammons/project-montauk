# Suggested Topic Files

Not every project needs all of these. Pick what fits. Create flat numbered files (`01-*.md`) or nest in subdirectories.

## Common topic files

| Suggested name | Covers |
|---|---|
| `01-methodology.md` | Core methodology / approach / algorithm |
| `02-view-specs/` | Per-screen or per-view specifications |
| `03-data-model.md` | Data entities, relationships, shape |
| `04-design-tokens.md` | Color, type, spacing, radii, shadows |
| `05-components.md` | Component inventory with file paths + variants |
| `06-motion-layout.md` | Motion rules, layout grid, breakpoints |
| `07-architecture.md` | System architecture, deployment, infra |
| `08-conventions.md` | Naming, file structure, import patterns |
| `09-terminology.md` | Domain vocabulary (also mirrored in spirit-memory/glossary.md) |
| `10-operational.md` | Day-to-day operational details, runbooks |
| `11-regulatory.md` | Compliance, legal, regulatory context |
| `12-quality-validation.md` | Testing, QA, validation approach |

## Not required

Start with what you actually need. Empty topic files waste reader attention. Add topic files only when you have content to put in them.
